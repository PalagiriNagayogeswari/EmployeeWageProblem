A Java project developed as part of the **Bridgelabz OOPS and DS** learning program. This repository demonstrates the step-by-step implementation of the Employee Wage Computation problem using Core Java concepts and Object-Oriented Programming principles.

- Java
- Object-Oriented Programming (OOP)
- Array

- Interfaces

## Use Cases

### UC1 - Check Employee Attendance
- Simulate employee attendance.
- Employee can be Present or Absent.

### UC2 - Calculate Daily Employee Wage
- Calculate daily wage for a full-time employee.

### UC3 - Add Part-Time Employee & Wage
- Introduce part-time employees.
- Calculate wages based on working hours.

### UC4 - Solve Using Switch Case
- Refactor attendance logic using the `switch` statement.

### UC5 - Calculate Wages for a Month
- Compute monthly wages based on working days.

### UC6 - Calculate Wages Till Maximum Working Hours
- Stop wage calculation when maximum working hours or working days are reached.

### UC7 - Refactor into Class
- Use class variables and class methods.
- Improve code organization.

### UC8 - Compute Employee Wage for Multiple Companies
- Calculate wages for multiple companies.
- Use method parameters for company-specific details.

### UC9 - Save Total Wage for Each Company
- Store company information using instance variables.
- Maintain total wage for each company.

### UC10 - Manage Employee Wage for Multiple Companies
- Create an `EmpWageBuilder` class.
- Store multiple `CompanyEmpWage` objects.

### UC11 - Interface Approach
- Implement wage computation using Java Interfaces.

### UC12 - Use ArrayList
- Replace arrays with `ArrayList`.
- Dynamically manage multiple companies.

### UC13 - Store Daily Wage
- Store daily wages in addition to total wages.
- Display day-wise wage details.

### UC14 - Query Total Wage by Company
- Retrieve total wage using company name.
- Search company details from the collection.
